### 6.2 关键修复详情

#### 修复1：BCrypt 密码哈希

**问题**：`init.sql` 中的 BCrypt 哈希值被截断为 21 字符（标准应为 60 字符）

**影响**：所有初始账号（admin/teacher/student）无法登录，报 `Invalid salt` 错误

**修复**：
```sql
-- 正确的 BCrypt 哈希（密码 123456）
UPDATE sys_user SET password = '$2b$10$FkI8HzXS1.D2dQ9Cq5KUB.1687oCT/xTWtcZUkZELuRvdVH3C7VOC';
```

#### 修复2：Classes 模块 500 错误

**问题**：`/classes/all` 接口返回 500，学生管理页面无法加载班级下拉列表

**根因**：`Classes` 实体有 `teacherName` 字段，但数据库 `class` 表没有 `teacher_name` 列

**修复**：
```java
// 1. 标记为非数据库字段
@TableField(exist = false)
private String teacherName;

// 2. 在 Service 中手动填充
public List<Classes> getAll() {
    List<Classes> list = baseMapper.selectList(null);
    list.forEach(c -> {
        if (c.getTeacherId() != null) {
            SysUser teacher = sysUserMapper.selectById(c.getTeacherId());
            c.setTeacherName(teacher != null ? teacher.getRealName() : "");
        }
    });
    return list;
}
```

#### 修复3：权限安全漏洞

**问题**：所有 Controller 接口无任何角色权限判断，学生可以调用任意接口

**影响**：严重安全漏洞，学生可以修改成绩、审批奖学金、删除用户等

**修复**：
```java
// 创建 SecurityUtils 工具类
public class SecurityUtils {
    public static void checkAdmin() { ... }
    public static void checkAdminOrTeacher() { ... }
    public static void checkSelfOrAdmin(Long userId) { ... }
}

// 在 Controller 中使用
@GetMapping
public Result<IPage<ScoreVO>> getPageList(ScoreQuery query) {
    SecurityUtils.checkAdminOrTeacher();  // 仅管理员和教师可访问
    return Result.success(scoreService.getPageList(query));
}
```

## 八、项目亮点

### 8.1 技术亮点

1. **前后端分离架构**：Vue 3 + Spring Boot 3，现代化技术栈
2. **JWT 认证**：无状态认证，支持分布式部署
3. **RBAC 权限模型**：基于角色的访问控制，权限粒度到接口级别
4. **AOP 日志记录**：自动记录所有敏感操作，便于审计
5. **Excel 导入导出**：Apache POI 实现批量数据处理
6. **数据可视化**：ECharts 实现 Dashboard 图表展示

### 8.2 代码亮点

1. **统一响应格式**：Result 类统一封装 API 响应
2. **全局异常处理**：GlobalExceptionHandler 统一处理异常
3. **参数校验**：JSR-303 注解实现请求参数校验
4. **代码复用**：Service 层抽象，减少重复代码
5. **边界情况处理**：空班级、新入学学生、多门挂科等场景覆盖

### 8.3 安全亮点

1. **密码加密**：BCrypt 加密存储，防止明文泄露
2. **SQL 注入防护**：MyBatis-Plus 参数化查询
3. **XSS 防护**：前端转义 + 后端校验
4. **权限校验**：后端主防线 + 前端辅助，双重保障
5. **操作日志**：敏感操作全部记录，可追溯
