## 任务清单

### 任务 1：确认本地代码已修复

**检查文件**：`backend/src/main/resources/application-dev.yml`

**确认 JDBC URL 正确**：
```yaml
url: jdbc:mysql://localhost:3306/student_management?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8&useUnicode=true
```

关键：`characterEncoding=UTF-8`（不是 utf8mb4）

### 任务 2：确认 application.yml（Railway 部署配置）

**文件**：`backend/src/main/resources/application.yml`

**确认内容**：
```yaml
spring:
  datasource:
    url: ${DATABASE_URL}
    driver-class-name: org.postgresql.Driver
```

Railway 用 PostgreSQL，通过环境变量 `DATABASE_URL` 连接 Neon 数据库。

### 任务 3：确认 pom.xml 依赖正确

**文件**：`backend/pom.xml`

**确认包含两个数据库驱动**（本地 MySQL + Railway PostgreSQL）：
```xml
<!-- MySQL（本地开发） -->
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>

<!-- PostgreSQL（Railway 部署） -->
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.7.3</version>
    <scope>runtime</scope>
</dependency>
```

### 任务 4：确认 Mapper XML 文件

**文件**：`backend/src/main/resources/mapper/*.xml`

**关键修改**：PostgreSQL 中 `class` 是保留字，表名需要加双引号：
```xml
<!-- 错误 -->
FROM class

<!-- 正确 -->
FROM "class"
```

涉及文件：
- DashboardMapper.xml
- RewardPunishMapper.xml
- 其他包含 class 表的 Mapper

## 常见问题排查

### Q1: Railway 部署失败

**检查**：
1. `Dockerfile` 是否在项目根目录
2. `pom.xml` 是否包含 PostgreSQL 驱动
3. 环境变量 `DATABASE_URL` 是否设置

### Q2: 登录返回 500 错误

**检查**：
1. Neon 数据库是否执行了 `init-pg.sql`
2. 环境变量 `DATABASE_URL` 是否正确
3. Railway 日志查看具体错误

### Q3: 前端访问报错

**检查**：
1. `frontend/.env.production` 的 API 地址是否正确
2. Vercel 构建日志是否有错误
3. 浏览器控制台网络请求是否成功

### Q4: 中文显示乱码

**检查**：
1. `application-dev.yml` 中 `characterEncoding=UTF-8`（本地）
2. PostgreSQL 连接字符串不需要额外编码参数
3. Neon 数据库默认使用 UTF-8
