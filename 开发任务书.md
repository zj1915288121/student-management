## 问题诊断

**错误信息**：`MyBatisSystemException: null`

**可能原因**：
1. Neon 数据库没有执行 `init-pg.sql`，表不存在
2. Neon 数据库没有执行 `test_data-pg.sql`，缺少测试数据
3. admin 用户不存在或数据不完整

---

## 修复步骤

### 步骤 1：检查 Neon 数据库表是否存在

打开 Neon SQL Editor（https://console.neon.tech → 你的项目 → SQL Editor），执行：

```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
```

**期望结果**：应该看到 11 张表：
- sys_user
- class
- student
- course
- score
- status_change
- reward_punish
- scholarship
- scholarship_apply
- sys_log

**如果没有这些表** → 执行步骤 2

**如果已经有表** → 执行步骤 3

---

### 步骤 2：执行建表脚本（如果表不存在）

在 Neon SQL Editor 中，复制并执行 `database/init-pg.sql` 的全部内容。

**文件位置**：`D:\Axm\Project\学生信息管理系统\database\init-pg.sql`

**执行方式**：
1. 用记事本或 VS Code 打开 `init-pg.sql`
2. 全选复制（Ctrl+A, Ctrl+C）
3. 粘贴到 Neon SQL Editor
4. 点击 Run

---

### 步骤 3：检查 admin 用户是否存在

在 Neon SQL Editor 中执行：

```sql
SELECT id, username, role, real_name, status FROM sys_user WHERE username = 'admin';
```

**期望结果**：
| id | username | role | real_name | status |
|:--:|:--------:|:----:|:---------:|:------:|
| 1 | admin | 1 | 系统管理员 | 1 |

**如果没有结果** → 执行步骤 4

**如果有结果但 real_name 是乱码或空** → 执行步骤 4

---
