# 阶段十：登录异常诊断与修复

## 项目目录
`D:\Axm\Project\学生信息管理系统`

## 问题描述
本地和部署环境登录均失败，POST `/auth/login` 返回：
```json
{"code": 500, "message": "系统异常，请联系管理员"}
```
`GlobalExceptionHandler` 捕获了 Exception 但未输出具体错误，导致真实原因被隐藏。

---

## 当前状态分析

### 1. 异常链路追踪
`GlobalExceptionHandler.handleException(Exception e)` 捕获所有未处理异常：
```java
log.error("系统异常：", e);
return Result.error(ResultCode.ERROR.getCode(), "系统异常，请联系管理员");
```
→ 只打印了日志但没把具体信息返回前端。

### 2. 登录流程中可能抛异常的位置
```
AuthController.login()
  → SysUserServiceImpl.login()
      → sysUserMapper.selectOne()          [数据库查询]
      → passwordEncoder.matches()          [BCrypt验证]
      → jwtUtils.generateToken()            [JWT生成]
```

### 3. application.yml 当前配置
```yaml
spring:
  datasource:
    url: ${DATABASE_URL}              # Railway/Neon环境变量
    username: ${DB_USER:postgres}
    password: ${DB_PASSWORD}
```
→ 本地运行需要设置 DATABASE_URL 环境变量

### 4. 已知问题（推测）
test_data-pg.sql 中的密码 BCrypt 哈希可能：
- 格式不标准（BCrypt $2a$ vs $2b$ 差异）
- 哈希值在 SQL 文件中被截断或格式错误
- 数据库中存储的密码字段为 NULL

## 修复任务

### 任务 1：增强 GlobalExceptionHandler，暴露真实错误（优先）
修改 `GlobalExceptionHandler.java`，让开发/测试环境下返回具体错误信息：

```java
@ExceptionHandler(Exception.class)
public Result<?> handleException(Exception e) {
    // 生产环境不暴露具体错误，开发/测试环境暴露
    String message = isDevProfile() ? e.getMessage() : "系统异常，请联系管理员";
    log.error("系统异常：", e);
    return Result.error(ResultCode.ERROR.getCode(), message);
}

private boolean isDevProfile() {
    Environment env = SpringContext.getBean(Environment.class);
    String[] profiles = env.getActiveProfiles();
    for (String p : profiles) {
        if ("dev".equals(p) || "test".equals(p)) return true;
    }
    String mode = System.getenv("APP_ENV");
    return "dev".equals(mode) || "local".equals(mode);
}
```

或者更简单，直接在 handleException 里同时打印和返回错误：
```java
@ExceptionHandler(Exception.class)
public Result<?> handleException(Exception e) {
    log.error("系统异常：", e);
    // 优先返回具体错误，堆栈信息放 data 字段方便调试
    String detail = e.getClass().getSimpleName() + ": " + e.getMessage();
    return Result.error(ResultCode.ERROR.getCode(), "系统异常：" + detail);
}
```

### 任务 3：根据真实错误修复
根据任务 2 的具体错误，进行针对性修复：

**情况 A**：BCrypt 哈希问题
→ 生成正确的 BCrypt 哈希，更新 init-pg.sql 中的管理员初始密码
→ 本地测试用 java -cp h2*.jar 或写个工具类生成 BCrypt 哈希

**情况 B**：数据库字段/表不存在
→ 检查 init-pg.sql 是否完整执行
→ 检查 entity 类字段和数据库表列是否匹配

**情况 C**：JWT 生成失败
→ 检查 jwtUtils.generateToken() 是否有空指针

**情况 D**：MyBatis-Plus 查询问题
→ 检查 mapper XML 和 entity 注解是否与 PostgreSQL 兼容

### 任务 4：确保本地能用 MySQL 启动
如果本地用的是 MySQL，需要配置本地 MySQL 连接。创建 `application-dev.yml`：
```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/student_management?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
```

### 任务 5：生成正确的测试数据密码哈希
写一个 Spring Boot 命令行工具或 main 方法，生成标准 BCrypt 哈希：
```java
// 运行一次生成哈希，然后更新 init-pg.sql
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String hash = encoder.encode("123456");
System.out.println("123456 的 BCrypt 哈希: " + hash);
```

### 任务 6：更新 init-pg.sql 和 test_data-pg.sql
用任务 5 生成的正确哈希替换 SQL 文件中的密码字段。
