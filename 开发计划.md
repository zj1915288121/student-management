## 四、数据库设计（核心表）

### ER 关系概览

```
student (学生) ──< score (成绩) >── course (课程)
    │
    ├──< scholarship_apply (奖学金申请)
    │
    ├──< reward_punish (奖惩记录)
    │
    └──< status_change (学籍异动)
```

### 核心数据表

| 表名 | 说明 | 主要字段 |
|-----|------|---------|
| `sys_user` | 用户表 | id, username, password, role, real_name |
| `student` | 学生表 | id, student_no, name, gender, class_id, major, phone |
| `class` | 班级表 | id, class_name, major, grade, teacher_id |
| `course` | 课程表 | id, course_name, credit, teacher_id, semester |
| `score` | 成绩表 | id, student_id, course_id, score, semester |
| `status_change` | 学籍异动表 | id, student_id, type, reason, status |
| `reward_punish` | 奖惩表 | id, student_id, type, content, date |
| `scholarship` | 奖学金表 | id, name, amount, level |
| `scholarship_apply` | 奖学金申请表 | id, student_id, scholarship_id, status |
| `sys_log` | 操作日志表 | id, user_id, action, time, ip |

## 六、接口设计规范

### RESTful API 规范

| 操作 | 方法 | URL 示例 |
|-----|------|---------|
| 查询列表 | GET | `/api/students?page=1&size=10` |
| 查询详情 | GET | `/api/students/{id}` |
| 新增 | POST | `/api/students` |
| 修改 | PUT | `/api/students/{id}` |
| 删除 | DELETE | `/api/students/{id}` |
| 批量删除 | DELETE | `/api/students/batch?ids=1,2,3` |
| 导入 | POST | `/api/students/import` |
| 导出 | GET | `/api/students/export` |

### 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```
