### 任务37：修复Classes模块500错误 ✅

**执行时间**：2026-04-19

**问题描述**：
- `/classes/all` 接口返回500错误，导致学生管理页面无法加载班级下拉列表
- 根因：Classes实体有 `teacherName` 字段，但数据库 `class` 表没有 `teacher_name` 列
- MyBatis-Plus查询时尝试映射不存在的列，抛出SQL异常

**修复操作**：
1. 给 Classes 实体的 `teacherName` 字段添加 `@TableField(exist = false)` 注解，标记为非数据库字段
2. 在 ClassesServiceImpl 中注入 SysUserMapper
3. 修改 `getAll()` 方法：查询班级列表后，遍历填充 teacherName（根据 teacherId 查询教师姓名）
4. 修改 `toVO()` 方法：分页查询时也自动填充 teacherName

**修改的文件**：
| 文件 | 修改内容 |
|-----|------|
| `Classes.java` | teacherName字段添加 `@TableField(exist = false)` |
| `ClassesServiceImpl.java` | 注入SysUserMapper，getAll和toVO中填充teacherName |

**验证**：`/classes/all` 返回200，班级数据正常显示

### 任务52：修复个人中心表单验证 ✅

**执行时间**：2026-04-19

**问题根因**：`SysUserController.updateProfile()` 使用了 `@Valid @RequestBody SysUserDTO dto`，而 `SysUserDTO` 中有 `@NotBlank username` 和 `@NotNull role` 校验，导致个人中心修改信息时触发不必要的校验。

**后端修复**：
1. 创建 `ProfileUpdateDTO.java`，只包含 `realName`/`phone`/`email` 三个字段，无必填校验
2. 修改 `SysUserController.updateProfile()` 使用 `ProfileUpdateDTO` 替代 `SysUserDTO`
3. 修改 `SysUserService.updateProfile()` 方法签名
4. 修改 `SysUserServiceImpl.updateProfile()` 方法签名

**前端修复**：
- 移除 `realName` 的 `required` 必填校验（姓名不应强制必填）
- 为手机号添加格式校验（`/^1[3-9]\d{9}$/`）
- 为邮箱添加格式校验（`type: 'email'`）
- 将 `prop="realName"` 改为 `prop="phone"` 和 `prop="email"` 以启用校验

**修改的文件**：
| 文件 | 修改内容 |
|------|---------|
| `backend/.../dto/ProfileUpdateDTO.java` | 新建，仅含 realName/phone/email |
| `backend/.../controller/SysUserController.java` | updateProfile 使用 ProfileUpdateDTO |
| `backend/.../service/SysUserService.java` | 方法签名改为 ProfileUpdateDTO |
| `backend/.../service/impl/SysUserServiceImpl.java` | 方法签名改为 ProfileUpdateDTO |
| `frontend/src/views/profile/index.vue` | 移除 realName 必填，添加 phone/email 校验 |

### 任务66：修复 JwtUtils 使用废弃 API ✅

**执行时间**：2026-04-19

5. 使用 `getPayload()` 替代 `getBody()`
6. 使用 `claims()/subject()/issuedAt()/expiration()` 替代 `setClaims()/setSubject()/setIssuedAt()/setExpiration()`

3. `DashboardMapper.xml`：`JOIN "class"` → `JOIN `class``，`FROM score WHERE score` → `` FROM `score` WHERE `score` ``
4. `RewardPunishMapper.xml`：`FROM "class"` → `` FROM `class` ``

**修改的文件**：
| 文件 | 修改内容 |

### 任务90：修复 application-dev.yml JDBC URL characterEncoding ✅

**执行时间**：2026-04-20

**问题描述**：
- `application-dev.yml` 中 JDBC URL 使用 `characterEncoding=utf8mb4`
- `utf8mb4` 是 MySQL 内部字符集名称，Java 的 `Charset.forName()` 不认识
- 导致 `UnsupportedEncodingException: utf8mb4`，JDBC 连接失败，所有登录返回 500

**修复操作**：
- 将 `characterEncoding=utf8mb4` 改为 `characterEncoding=UTF-8`
- `UTF-8` 是 Java 标准字符集名称，MySQL JDBC 驱动会自动映射到 MySQL 的 `utf8mb4`

**修改的文件**：
| 文件 | 修改内容 |
|-----|------|
| `backend/.../resources/application-dev.yml` | `characterEncoding=utf8mb4` → `characterEncoding=UTF-8` |
